极致轻量

Chapter 2

安装 Alpine Linux

Alpine 全系 3.x 版本的安装流程完全一致。标准镜像约 200MB，virt 镜像约 50MB。

下载镜像

从 alpinelinux.org/downloads 选择适用于你架构的镜像。

bash
# x86_64
wget https://dl-cdn.alpinelinux.org/alpine/latest-stable/releases/x86_64/alpine-standard-{{version}}-x86_64.iso

# ARM64 (树莓派等)
wget https://dl-cdn.alpinelinux.org/alpine/latest-stable/releases/aarch64/alpine-standard-{{version}}-aarch64.iso

启动并安装

从 ISO 启动后，登录 root（空密码），执行 setup-alpine 按向导完成配置——键盘布局、主机名、网卡、时区、源、ssh、磁盘、用户。

bash
localhost:~# setup-alpine
localhost:~# reboot

下载 virt 镜像

bash
wget https://dl-cdn.alpinelinux.org/alpine/latest-stable/releases/x86_64/alpine-virt-{{version}}-x86_64.iso

创建虚拟机

bash
qemu-img create -f qcow2 alpine.qcow2 4G
qemu-system-x86_64 -m 512 -cdrom alpine-virt-{{version}}-x86_64.iso -drive file=alpine.qcow2,format=qcow2 -net nic -net user

安装后配置

安装完成后建议立即执行：

bash
apk update && apk upgrade
apk add bash curl wget vim tmux
setup-timezone -z Asia/Shanghai # 设置时区
setup-sshd -c openssh # 启用 SSH

apk 包管理

删除

📋

列出已安装

📜

详细信息

⚡

升级

apk update && apk upgrade

bash
# 常用操作
apk update # 更新源缓存
apk upgrade # 升级所有包
apk add bash curl wget vim htop # 安装包
apk del vim # 删除包
apk search nginx # 搜索
apk info # 列出已安装
apk info -a curl # 查看详细信息
apk cache clean # 清理缓存

# 安装编译工具
apk add build-base gcc g++ make

# 安装 Docker
apk add docker docker-compose
⚠️
musl vs glibc 兼容：Alpine 使用 musl libc 替代 glibc。部分闭源软件可能不兼容。如果遇到缺失 glibc 符号的错误，安装兼容层：apk add gcompat。
但 GOST 本身是 Go 静态编译的二进制，不依赖 glibc，在 musl 下完美运行。
Chapter 4

OpenRC 服务管理

Alpine 全系使用 OpenRC 替代 systemd。核心命令比 systemd 更少，5 分钟即可上手。

操作systemd 命令OpenRC 命令说明
启动服务systemctl start nginxrc-service nginx start立即启动
停止服务systemctl stop nginxrc-service nginx stop立即停止
重启服务systemctl restart nginxrc-service nginx restart停止再启动
查看状态systemctl status nginxrc-service nginx status显示运行/停止/crashed
开机自启systemctl enable nginxrc-update add nginx default加入默认运行级
取消自启systemctl disable nginxrc-update del nginx从运行级移除
重载配置systemctl reload nginxrc-service nginx reload发送 SIGHUP
查看日志journalctl -u nginxcat /var/log/messages系统日志
列出服务systemctl list-unitsrc-service --list列出所有服务
重载服务定义systemctl daemon-reload（不需要）OpenRC 每次读取文件

OpenRC 服务脚本编写

简单服务用变量声明即可，不需要写 start/stop 函数：

/etc/init.d/myapp
#!/sbin/openrc-run
name="myapp"
description="My Custom Service"
command="/usr/local/bin/myapp"
command_args="--config /etc/myapp/config.yml"
command_background=true
pidfile="/run/${name}.pid"

depend() {
need net
}

GOST 节点服务的 OpenRC 脚本（install.sh 自动生成）：

/etc/init.d/gost
#!/sbin/openrc-run
name="gost"
description="Gost Proxy Service"
command="/etc/gost/gost.sh"
command_background=true
pidfile="/run/gost.pid"

depend() {
need net
}

启动包装脚本（解决 daemonize 后 CWD 被重置的问题）：

/etc/gost/gost.sh
#!/bin/sh
cd /etc/gost && exec /etc/gost/gost "$@"

网络配置

Alpine 使用 /etc/network/interfaces 配置网络，风格与 Debian 一致。

/etc/network/interfaces
# 环回接口
auto lo
iface lo inet loopback

# DHCP
auto eth0
iface eth0 inet dhcp

# 静态 IP
# auto eth0
# iface eth0 inet static
# address 192.168.1.100
# netmask 255.255.255.0
# gateway 192.168.1.1
# dns-nameservers 1.1.1.1 8.8.8.8
bash
# 重启网络
rc-service networking restart

# 查看 IP
ip addr show

# 添加 DNS
echo "nameserver 1.1.1.1" > /etc/resolv.conf

# 开放端口
apk add iptables
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A INPUT -p tcp --dport 6365 -j ACCEPT
rc-service iptables save
Chapter 6

Docker on Alpine

Alpine 是 Docker 最流行的宿主机操作系统之一，安装配置在所有 Alpine 3.x 上一致。

安装 Docker

bash
apk add docker docker-compose
rc-update add docker default
rc-service docker start

验证

bash
docker info
docker run --rm hello-world

配置 IPv6

如需 Docker 容器使用 IPv6：

bash
cat > /etc/docker/daemon.json <<'EOF' { "ipv6": true, "fixed-cidr-v6": "fd00::/80" } EOF rc-service docker restart
💡
注意：Alpine 上重启 Docker 使用 rc-service docker restart。Flux Panel 的 panel_install.sh 已自动处理此差异——在 Alpine 上检测到 is_alpine 后优先使用 rc-service docker restart。
Chapter 7

部署 GOST 节点

在 Alpine 上一键部署 Flux Panel 的 GOST 代理节点。整个过程只需要 install.sh 一个文件。

✅
跨发行版兼容：同一个 install.sh 同时支持 Alpine Debian Ubuntu CentOS Fedora Arch openSUSE 等。脚本自动检测系统类型，Alpine 用 OpenRC，其他用 systemd。

安装依赖

Alpine 上只需三个包：

bash
apk add bash curl coreutils dos2unix

Debian/Ubuntu: apt install bash curl  |  CentOS: yum install bash curl

上传 install.sh

从本机传到 Alpine 服务器：

bash
# 本机执行
scp install.sh root@你的节点IP:/root/

运行安装

SSH 上服务器，确保行尾正确后运行：

bash
dos2unix install.sh # 如从 Windows 上传，需要转 LF
bash install.sh -a 面板地址:端口 -s 密钥

脚本会自动：检测系统类型 → 下载对应架构的 GOST 二进制 → 创建配置文件 → 注册服务（OpenRC 或 systemd）→ 启动并设为开机自启。

验证节点状态

bash
# 服务状态（OpenRC / systemd 自动适配）
rc-service gost status # Alpine
systemctl status gost # Debian/Ubuntu/CentOS

# 进程检查
ps aux | grep gost

# 手动前台测试（可以看到实时输出）
cd /etc/gost && timeout 5 ./gost

检查面板连通性

bash
nc -zv 面板地址 端口
# 或
curl -I http://面板地址:端口
⚠️
常见失败原因：
1. CRLF 行尾 — 从 Windows 上传后必须 dos2unix install.sh，否则报 $\r 错误。
2. 工作目录问题 — OpenRC 的 start-stop-daemon --background 后台化时会 chdir 到 /，导致 GOST 找不到 /etc/gost/config.json。install.sh 通过一个包装脚本 /etc/gost/gost.sh 解决此问题（先 cd 再 exec）。
3. 网络不通 — 检查是否能连到面板地址和端口。

Chapter 8

兼容性一览

install.sh 是一个跨发行版、跨架构的通用安装脚本，在同一个文件里同时处理了 OpenRC 和 systemd 两条路径。

项目兼容说明
Alpine 3.10 ~ 3.21✓脚本仅检测 ID=alpine，不检查版本号。全系 OpenRC + apk。
Alpine Edge✓Edge 同样使用 OpenRC，完全兼容。
Debian / Ubuntu✓走 systemd 分支 + apt。tcpkill 安装 apt install dsniff。
CentOS / RHEL / Fedora✓走 systemd 分支 + yum/dnf。tcpkill 安装 yum/dnf install dsniff。
Arch Linux / Manjaro✓走 systemd 分支 + pacman。
openSUSE / SLES✓走 systemd 分支 + zypper。
Gentoo✓走 systemd 分支 + emerge。
macOS部分仅 tcpkill 安装走 Homebrew，GOST 服务不受支持。
x86_64 (amd64)✓最广泛支持，自动下载 amd64 版本 GOST 二进制。
ARM64 (aarch64)✓树莓派、Oracle ARM、AWS Graviton 等。自动识别并下载 arm64 版本。
ARM32 / 其他架构fallback脚本 fallback 下载 amd64，但无法执行。需自行编译对应架构的 GOST。

检测逻辑源码

install.sh 中用于跨平台适配的核心代码：

install.sh 检测代码
# 检测是否为 Alpine Linux（不检查版本号）
is_alpine() {
if [ -f /etc/os-release ]; then
. /etc/os-release
[ "$ID" = "alpine" ]
return $?
fi
return 1
}

# 检测系统架构（自动适配 x86_64 / ARM64）
get_architecture() {
ARCH=$(uname -m)
case $ARCH in
x86_64) echo "amd64" ;;
aarch64|arm64) echo "arm64" ;;
*) echo "amd64" ;;
esac
}

# 安装 tcpkill（常见发行版全覆盖）
case $DISTRO in
ubuntu|debian) apt install -y dsniff ;;
centos|rhel|fedora) yum/dnf install -y dsniff ;;
alpine) apk add dsniff ;;
arch|manjaro) pacman -S dsniff ;;
opensuse*|sles) zypper install -y dsniff ;;
gentoo) emerge net-analyzer/dsniff ;;
esac
✅
核心设计原则：不要检测"不是 Alpine"，而是检测"是 Alpine"——{ }is_alpine() 返回 true 就走 OpenRC，返回 false 就走 systemd。这样新增任何未来的发行版都不需要改脚本。

故障排查

GOST 节点部署和 Alpine 使用中的常见问题及解决方法。

症状原因解决
rc-service gost status 显示 crashedconfig.json 未加载到运行 cd /etc/gost && timeout 5 ./gost 看报错。通常是工作目录不对（OpenRC daemonize 后 chdir 到 /），但 install.sh 已通过 gost.sh 包装脚本解决此问题。
面板显示节点离线WebSocket 连接失败nc -zv 面板地址 端口 检查连通性，curl -I http://面板地址:端口 检查 HTTP 响应。
上传后运行报 $\rWindows CRLF 行尾dos2unix install.sh 或 sed -i 's/\r//' install.sh
ps aux | grep gost 看不到进程服务启动后立刻退出cat /var/log/messages | grep gost 查看日志。
安装时报 "No such file"脚本行尾是 CRLFdos2unix install.sh
手动 cd /etc/gost && ./gost 成功，但服务启动就 crashOpenRC 后台化时重置了 CWD确认 /etc/init.d/gost 中 command 指向的是 /etc/gost/gost.sh 而非 /etc/gost/gost
症状原因解决
ping 不通外网DNS 或路由未配置检查 /etc/resolv.conf，添加 nameserver 1.1.1.1
端口无法访问防火墙未放行iptables -A INPUT -p tcp --dport 端口 -j ACCEPT
DHCP 获取不到 IP网卡未正确初始化setup-interfaces 重新配置或手动编辑 /etc/network/interfaces
IPv6 不工作内核模块未加载modprobe ipv6
Docker 网络不通iptables 规则被覆盖rc-service docker restart
症状原因解决
apk add 报签名错误源证书过期apk update && apk upgrade
找不到某个包testing 源未启用添加 edge/testing 到 /etc/apk/repositories
glibc 程序报错musl 不兼容apk add gcompat
apk search 结果太少仅启用了 main 源启用 community 源：echo "https://dl-cdn.alpinelinux.org/alpine/v3.21/community" >> /etc/apk/repositories
症状原因解决
SSH 连不上openssh 未启动apk add openssh && rc-update add sshd default && rc-service sshd start
时区不对未设置时区setup-timezone -z Asia/Shanghai
磁盘空间满日志或缓存堆积apk cache clean && rm -rf /var/log/*.gz
文件系统只读磁盘异常挂载mount -o remount,rw /
busybox 命令不够用需要 GNU 版本apk add coreutils findutils grep sed

一键诊断脚本

在 Alpine 服务器上运行，快速定位问题：

bash
#!/bin/sh
echo "=== 系统 ===" && uname -a
echo "=== 发行版 ===" && cat /etc/os-release | grep -E "^(ID|VERSION_ID)="
echo "=== 内存 ===" && free -h | head -2
echo "=== 磁盘 ===" && df -h / | tail -1
echo "=== GOST ===" && (rc-service gost status 2>&1 || systemctl status gost 2>&1 || echo "not installed")
echo "=== 进程 ===" && (ps aux | grep gost | grep -v grep || echo "no gost process")
echo "=== 配置 ===" && ls -la /etc/gost/ 2>&1
echo "=== 连通性 ===" && nc -zv 面板地址 端口 2>&1
